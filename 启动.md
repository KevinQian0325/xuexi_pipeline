# 启动和新设备环境配置

## 1. 本机日常启动

终端 1：启动后端。

```bash
conda activate xuexi-pipeline
cd "/Users/kevinqian/Documents/Code/Intern XuanRong Technology/xuexi_pipeline"
python -m uvicorn backend.api_server:app --host 127.0.0.1 --port 8000
```

终端 2：启动前端。

```bash
cd "/Users/kevinqian/Documents/Code/Intern XuanRong Technology/xuexi_pipeline/frontend"
npm run dev
```

浏览器打开前端提示的地址，通常是：

```text
http://localhost:5173
```

## 2. Windows 新设备环境配置

以下命令默认在 Windows 的 PowerShell 或 Anaconda Prompt 中执行。新设备第一次配置时先完成本节，之后日常使用看“Windows 日常启动”。

先安装这些基础软件：

```text
Miniconda 或 Anaconda
Node.js LTS
Git
ffmpeg
```

建议用 Anaconda Prompt 执行 Python 相关命令。

进入项目目录：

```powershell
cd "D:\你的路径\xuexi_pipeline"
```

创建并激活 Python 环境：

```powershell
conda create -n xuexi-pipeline python=3.12 -y
conda activate xuexi-pipeline
```

安装后端依赖：

```powershell
pip install -r requirements.txt
```

安装 Playwright 浏览器：

```powershell
python -m playwright install chromium
```

安装前端依赖：

```powershell
cd frontend
npm install
cd ..
```

安装 ffmpeg 后，确认命令可用：

```powershell
ffmpeg -version
```

如果提示找不到 `ffmpeg`，说明还没有把 ffmpeg 加入 Windows 环境变量 `Path`。

第一次启动后，在前端页面点击“密钥配置”，填写：

```text
XUEXI_APP_ID
XUEXI_ACCESS_TOKEN
结果文件存储根目录
```

这些配置会保存到 `程序运行文件夹\app.db`，项目现在不依赖 `.env`。

## 5. 常见问题

如果提示找不到 `conda`，请使用 Anaconda Prompt，或确认 Miniconda/Anaconda 已正确安装。

如果提示找不到 `npm`，请安装 Node.js LTS，并重新打开终端。

如果视频下载、音频提取失败，先检查：

```powershell
ffmpeg -version
```

如果网页 JSON 捕获失败，先检查 Playwright Chromium 是否安装：

```powershell
python -m playwright install chromium
```

如果端口被占用：

```powershell
netstat -ano | findstr :8000
netstat -ano | findstr :5173
```

可以关闭占用端口的程序后再重新启动。
